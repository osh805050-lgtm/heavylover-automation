## 본질 원인 2가지

### 1. 박제된 수치 = 현재 사실로 착각
- bridge_10b.json·unit_economics.json·CLAUDE.md는 **작성 시점의 사실**이지 **현재 사실 아님**
- 박재영 인건비, 고정비, 광고비, 결제 퍼널 상태 모두 동일 패턴
- → **§수치현행성검증** 카테고리 신설

### 2. 본문 언급 = 제출 요건으로 추정
- 사업계획서 본문에 "KOTRA 합격" 언급 있다는 이유로 제출 서류에 자동 추가
- 공고문 원본 확인 없이 논리로 추정
- → **§제출요건정본확인** 카테고리 신설
