# 2026-04-28 — SS 누락 사고 + 자동배포 구축

> 종합 인시던트 리포트. 시간순 흐름 + 원인 + 영구 해결책 + 검증 결과.

## TL;DR

- **사고**: 4/27(월) 11시 발주 자동화에서 스마트스토어 주문 33건 누락 (그중 26건이 4/28 발주 시점까지 미발송, 7건은 취소 요청 진행 중)
- **근본 원인 2가지**:
  1. SS 조회가 `hours_back=24` 변경 윈도우 + 평일(`1-5`) cron 갭 → 금~일 결제분 영구 누락
  2. Vultr 서버가 GitHub clone이 아닌 **단순 복사 폴더**라 코드 갱신이 매번 수동 SCP 의존
- **영구 해결**:
  1. SS: `orders_pending_dispatch(days_back=14)` 신설 — 14일 PAYED 전수 + dedupe + 상세 후 재필터
  2. 서버: `.git` 이식 + `deploy-vultr.yml` 워크플로우 — `git push`만 하면 SSH→git pull→텔레그램 알림 자동
- **추가 발견**: `is_shipping_overdue`가 `placeOrderStatus`를 안 봐 취소 요청 7건을 "발송기한초과"로 오판정 → `OK` 체크 추가
