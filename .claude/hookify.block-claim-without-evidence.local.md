⛔ **증거 없는 완료 선언 차단** — 보고 순서 강제

## 핵심 원칙

검증 결과(명령 + 출력)를 **먼저** 보여주고, **그 다음** 통과 선언.

❌ "통과 확인했습니다. 다음으로..."
✅ "$ pytest → 24 passed in 2.04s ← 증거 / 통과 확인. 다음으로..."

## 면제 조건 (응답 본문에 증거 있으면 차단 해제)

같은 응답 안에 다음 중 **하나 이상** 있으면 통과:

1. **명령어 출력 증거**: 코드블록(``` ``` 또는 인라인) 안에 다음 패턴
   - `passed` / `failed` / `FAIL` / `exit 0` / `All checks passed`
   - `✅` + 숫자(예: `✅ 24 passed`)
   - `❌` + 에러 메시지
   - `$` 또는 `>` 프롬프트로 시작하는 명령
   - HTTP 상태 (`200 OK`, `401`, `429` 등)
   - pytest/ruff/curl/python/gh 실행 결과 trace

2. **직전 Bash 도구 호출 trace** (transcript 검증): 같은 응답 또는 직전 응답에서 Bash 도구 호출 후 stdout/stderr 확인 흔적

3. **검증 불가 명시**: "검증 불가 사유: [구체적]" — 단 "그냥", "별 거 아님" 같은 모호한 사유는 면제 X

## 차단 대상

- "통과 확인" / "완료 보고" / "수정했습니다" / "작동합니다" 어휘
- + 같은 응답에 명령어 출력 증거 0건
- + 직전 Bash trace 0건
- + 검증 불가 사유 미명시
- → **차단**

## 대체 행동

응답 다시 작성:

1. **먼저** 검증 명령 실행 (Bash/pytest/ruff/python/curl/gh):
   ```
   $ python -m pytest tests/...
   24 passed in 2.04s
   ```
2. **그 다음** 결과 해석:
   ```
   ✅ 24/24 통과. 회귀 없음.
   ```
3. **마지막** 선언:
   ```
   통과 확인.
   ```

순서 위반 시 다시 작성.

## 면제 케이스 (차단 안 됨)

- "다음 단계로 넘어가겠습니다" (선언 아님)
- "계획 검토 완료, 작성 시작하겠습니다" (작업 진행 의사)
- 단순 보고: "현재 진행 상황 보고합니다" (선언 어휘 없음)
- 검증 불가 작업의 정직한 보고: "Vultr SSH 접근 못해서 실제 실행 검증 불가 — 다음 cron 실행 결과 대기"

## 비전공자 안내 (차단 시 표시)

"통과했다고 하기 전에 어떤 명령을 돌려서 어떤 결과가 나왔는지 먼저 보여주세요. 결과 없이 '통과'만 적으면 막힙니다. 형식: 명령어 → 출력 → 그 다음 결론."
