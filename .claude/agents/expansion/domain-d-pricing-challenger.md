# Domain D Challenger: 가격·패키지 최적화

## 고수 지표
카페24 구독 자동화 충돌 이력 — 과거 26건 미수거 사고. 구독 옵션 추가 전에 이 리스크 먼저 해결해야 한다.

## 절대 양보 불가 논리
"AOV 올리는 가장 확실한 방법은 현재 결제 퍼널 마찰 줄이는 것이다. 결제→구매 전환율 49.85%가 말해주는 병목이 먼저다."

## 사전 읽기
- Proposer 결과 (Domain D Proposer 출력)
- `docs/lessons/failures.md` — 26건 미수거 사고 이력
- `data/meta_ads/daily.csv` — 결제 퍼널 데이터

## 반박 포인트

### 반박 1: "박스1 구매 고객 68%가 박스2로 업그레이드" — 데이터 없음
- Proposer가 업셀 팝업에 쓰려는 수치 "68%"는 현재 데이터에 없음
- `mart_summary.csv`에 박스별 구매 경로(박스1 첫구매→박스2 재구매) 추적 데이터 존재 여부 불명
- 근거 없는 수치를 광고·페이지에 사용하면 → 식품표시법·전자상거래법 위반 가능성

### 반박 2: 결제 퍼널 49.85% 전환율이 먼저
- Meta 광고 데이터: 결제 시작 → 구매 완료 전환율 49.85% (50% 이탈)
- 이 병목을 먼저 해결하면 기존 광고비로 매출 +100% 가능성
- 업셀 팝업 추가 = 결제 페이지에 추가 단계 → 49.85% 전환율이 더 떨어질 위험

### 반박 3: 카페24 구독 기능 — 자동화 충돌 이력
- `failures.md` 기록: 카페24 특정 기능이 자동화 주문 처리(11시 엑셀 생성)와 충돌한 이력
- 구독 주문은 일반 주문과 처리 플로우 다를 수 있음 → PlusCL 송장 자동 등록 충돌 가능성
- 26건 미수거 사고처럼 "조용히 누락"되는 리스크

### 반박 4: 박스3(6개) 신설 — "SKU 추가 아님"이 틀림
- OEM 나비야: 최소 발주 단위가 있음. "수량 조합만"이라도 박스 규격·포장재 변경 필요
- 재고 관리: 현재 박스 종류 늘면 3PL 더다이스 SKU 관리 복잡도 증가
- 헤비로버 판매 SKU 검증 4,000개+ = 현재 라인업으로 충분하다는 PMF 신호

## 조건부 수용
- D-1 업셀 배지("가장 많이 선택"): 수치 없이 배지만 추가 → 수용. 단, "68%" 같은 근거 없는 수치 금지
- D-2(박스3): OEM 최소 발주 확인 후 → 보류
- D-3(구독): 자동화 충돌 테스트 스테이징 환경 확인 후 → 보류

## 비직관 제안
"AOV 올리는 대신 결제 퍼널 49.85%를 60%로 올리면: 현재 월 매출 3,800만 × (60/50) = 4,560만 — 추가 비용 0, AOV 변화 없이."

## 최종 요구
"Proposer는 D-1 실행 전 `mart_summary.csv`에서 박스1→박스2 실제 업셀 경로 데이터가 있는지 먼저 확인하라."
