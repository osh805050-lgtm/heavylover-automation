# Domain B Challenger: CRM / 재구매 트리거

## 고수 지표
M+1 14%의 원인 5가지 가설 중 어느 것인지 모른다 — 원인 모르고 트리거 이메일 보내는 것은 가설 검증이 아니라 희망 발사.

## 절대 양보 불가 논리
"H4 효과가 있는지 없는지를 판단할 측정 인프라(그룹 태깅)가 먼저다. 이메일 먼저 보내고 나중에 측정하면 결과 해석 불가."

## 사전 읽기
- Proposer 결과 (Domain B Proposer 출력)
- `data/analysis_10b/sheets/mart_cohort.csv`
- `docs/analysis_10b/02_cohort_paradox.md` — M+1 원인 가설 5개

## 반박 포인트

### 반박 1: M+1 14% 원인이 Day 10 트리거 부재라는 근거 없음
- `02_cohort_paradox.md`의 가설 5개: H1(상품 만족도) / H2(배송 경험) / H3(재구매 트리거 부재) / H4(P50 타이밍) / H5(SKU 권태)
- Proposer는 H3+H4가 원인이라 가정하지만 데이터 검증 없음
- H1(상품 자체 문제)이 원인이면 이메일 아무리 보내도 M+1 불변

### 반박 2: 카페24 이미 74% 재구매율 — 이미 작동 중인 것이 있다
- 카페24 누적 재구매율 74% (고객마스터 기준)
- "이미 재구매 잘 하는 고객 풀이 있는데 CRM이 없어도 작동 중" → CRM 추가 효과가 얼마나 될지 불명
- 제안 B-1의 "M+1 14→17~18%"는 타 D2C 케이스 참고 — 헤비로버 적용 가능성 검증 없음

### 반박 3: 그룹 태깅 B-3 없이 B-1 먼저 보내면 측정 불가
- Proposer도 인정: B-3(그룹 태깅)이 B-1과 동시에 필요
- 그런데 B-3 구현이 Claude 작업 2~3시간 + 카페24 API 연동 테스트 = 최소 3~5일
- "즉시 가능"이라는 B-1 주장은 B-3 없이는 성립 안 함

### 반박 4: 카페24 이메일 기능 실제 자동화 트리거 확인 필요
- 카페24 이메일이 "구매 후 N일 자동 발송" 기능 제공하는지 실제 확인 필요
- 기능 없으면 별도 ESP(스티비/메일침프) 연동 필요 — 추가 비용 월 3~10만원 + 세팅 2주

## 조건부 수용
- B-3(그룹 태깅 인프라) 먼저 구축 → B-1 실행 → 측정: 이 순서면 수용
- "B-1 즉시, B-3 나중" 순서는 거부 — 측정 못 하는 실험은 실험이 아님

## 최종 요구
"Proposer는 카페24 마케팅 이메일 기능에서 '구매 후 N일 자동 발송' 가능 여부를 먼저 확인하라. 그게 안 되면 B-1은 즉시 가능 제안이 아니다."
